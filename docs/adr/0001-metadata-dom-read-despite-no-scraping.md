# 表示用メタ情報は「閲覧専用」方針下でも DOM から取得する

/ status: accepted

本ツールは利用規約への配慮から「自動操作・スクレイピングは実装しない」(仕様書 13.1) を建前とするが、Column ヘッダーとサイドバーに表示する `username` / `avatarUrl` を、ログイン成功検知時に `executeJavaScript` でページ内 DOM から限定的に読み取る（案2）。

## 背景・代替案

- **案1（取得なし）**: `displayName`（ユーザー手入力）のみで UI を構成し、DOM には一切触れない。規約・セキュリティと完全整合するが、アバターが出ず UX が貧弱。
- **案2（採用）**: 表示用メタ情報のみを DOM から読み取る。投稿・操作の自動化は一切行わない。UX は良いが、(a) 13.1 の「スクレイピングしない」という対外的建前を「自動投稿はしないが、表示用メタ情報の読み取りは行う」に緩める必要があり、(b) サービスの UI 変更で読み取りが壊れる保守コストが発生する。

## 帰結

- 取得は**ベストエフォート**。失敗時は `username`/`avatarUrl` は null のままとし、UI は `displayName` で成立させる。
- `username` 未取得時はプロフィール遷移メニューをグレーアウトする（username を必要とするため。ADR とは別に仕様で規定）。
- 読み取りに使う JS は固定文字列のみとし、ユーザー入力を文字列に埋め込まない。
